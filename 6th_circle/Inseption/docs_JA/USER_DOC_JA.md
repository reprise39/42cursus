# 利用者マニュアル - Inception

このドキュメントでは、Inceptionそのものについて及び利用方法ついて説明します。

## 1. Inseptionとは（提供されるサービス）
以下のサービスで構成される安全なWordPressサイトを提供します。
- **NGINX**: インフラの入り口。ポート443を介してHTTPS通信を提供します。
- **WordPress**: コンテンツ管理システム（CMS）。PHP-FPMによって動作します。
- **MariaDB**: WordPressのデータを保存するためのデータベース。

## 2. プロジェクトの開始と停止
インフラを起動するには、プロジェクトのルートディレクトリで以下を実行します：
`make`

データを保持したままサービスを停止するには：
`make stop`

## 3. ウェブサイトへのアクセス方法
### 🌐 一般サイト
ブラウザを開き、以下のURLにアクセスしてください：

WordPressの管理を行うには、以下からログインしてください：

> **注意:** 自己署名証明書を使用しているため、ブラウザに「保護されていない通信」という警告が出ます。<br>
> こと今回においては問題ないはずなので「詳細設定」から「アクセスする」を選択して進んでください。

## 4. 認証情報（ログイン情報）の管理
データベースやWordPressのパスワード類は `secrets/` 以下の.txtファイルに保存されています。
ログイン名などは公開情報として`srcs/.env`に記載されています。

> **注意:** これらの値はmake時にセットされます。運用途中で変えても変更できませんのでむやみに変更しないでください。<br>
> （わからなくなってしまいます。）

## 5. 動作確認
すべてのサービスが正しく動いているか確認するには、以下を入力してください：
`docker ps`

`nginx`, `wordpress`, `mariadb` のすべてのステータスが **Up** になっていれば正常です。
